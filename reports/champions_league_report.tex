\section{Introduction}
This project aims to build a comprehensive dataset of UEFA Champions League matches from 2011 to 2025. The dataset includes technical statistics, tactical lineups, and official match details to enable advanced sports performance analytics.

\section{Data Engineering Pipeline}
The pipeline is designed to be incremental and robust, merging data from multiple official and semi-official sources.

\subsection{Sources}
\begin{itemize}
    \item \textbf{UEFA Official API:} Used for match officials, kickoff times, and official lineups.
    \item \textbf{ESPN Summary API:} Used for granular match statistics (possession, shots, fouls) and event timelines.
    \item \textbf{Candidate enrichment sources:} BALLDONTLIE UCL, football-data.org, openfootball, API-Football, Statorium, and Data Sports Group are tracked as source candidates for missing-field cross-reference. Sources that require credentials or commercial licensing are not used unless access is explicitly available.
    \item \textbf{Generic file ingestion:} Local source drops can be read from CSV, TSV, Excel, JSON, JSONL, HTML tables, Parquet, ZIP archives, directories, and plain text.
    \item \textbf{Storage Strategy:} The pipeline outputs cleaned datasets in both \textbf{CSV} (for accessibility) and \textbf{Parquet} (for high-dimensional analysis). Parquet is the primary input for the PCA tactical engine.
    \item \textbf{Manual Overrides:} A JSON-based cache handles edge cases where automated discovery fails due to team name variations.
\end{itemize}

\subsection{Data Fields}
The following fields are extracted and validated:
\begin{itemize}
    \item \textbf{Structural:} Season, fase, jornada, fecha, local, visitante, resultado, etc.
    \item \textbf{Officials:} Principal referee and linesmen.
    \item \textbf{Tactical:} Starting lineups (planteles) and coaches.
    \item \textbf{Statistical:} Total shots, shots on target, possession \%, fouls, and corners.
\end{itemize}

\section{Data Quality and Validation}
A diagnostic suite has been implemented to ensure data integrity and track coverage.

\subsection{Coverage Report}
Current quality reporting is generated by \texttt{logs/data\_quality\_report.json}. The report enforces two gates: each column must have at most 0.25\% missing values, and the total processed data must reach the 1.5M-record requirement. These gates are not satisfied by duplicating rows or fabricating records; missing values above the threshold must first be cross-referenced against additional sources.

\section{Conclusion}
The ingestion pipeline is operational and now writes both CSV and Parquet outputs. The project has expanded to include a \textbf{Principal Component Analysis (PCA)} module that processes 4,065 player-season records across 33 tactical features. The next milestone is source expansion to satisfy the 1.5M-record gate and documented imputation, following missing-data literature such as Little and Rubin, van Buuren, and football-specific per-90 normalization methods.
